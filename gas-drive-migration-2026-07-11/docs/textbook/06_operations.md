# 第6章 運用ガイド — 制限事項・トラブルシューティング・FAQ

[← 第5章](./05_dev_environment.md) | [目次](./README.md) | [付録: 用語集 →](./99_glossary.md)

## 6.1 どれくらい時間がかかるか

目安: **ファイル1件の移動 ≈ 0.5〜1秒** + フォルダごとの一覧取得。
1万ファイルなら実処理は2〜3時間程度で、これを 4.5分の実行 + 1分の待機の
サイクルで自動消化していく。人手は不要。

### 関係する GAS / Google のクォータ (無料 Google アカウントの場合)

| 制限 | 値 (目安) | 本ツールでの対策 |
| --- | --- | --- |
| 1回のスクリプト実行時間 | 約 6 分 | 4.5分で自動中断 → トリガーで自動再開 |
| トリガーによる合計実行時間 | 90 分/日 | 超大規模移行では尽きることがある。翌日自動的に回復するのを待つか、**メニュー「中断からの再開」を手動実行**すれば消費せず進められる |
| スクリプトプロパティ容量 | 値 9KB / 全体 500KB | チャンク分割保存 (第4章)。それでも溢れる規模はサブフォルダごとに分割実行 |
| Drive API 呼び出しレート | 短時間に集中すると 429 | 指数バックオフでリトライ |

<details>
<summary>📖 用語解説: クォータ (quota)</summary>

サービスの使いすぎを防ぐための利用上限。GAS では「1日あたりの合計実行時間」
「メール送信数」など多数のクォータがあり、超えると一時的にエラーになる。
多くは24時間で回復する。Google Workspace アカウントは無料アカウントより
上限が緩い。

</details>

### 移行先 (共有ドライブ) 側の上限

| 制限 | 値 |
| --- | --- |
| 共有ドライブ1つあたりのアイテム数 | 400,000 (ファイル+フォルダ) |
| フォルダの入れ子の深さ | 100 階層 |
| 1日にアップロード/取り込みできる量 | 750GB (移動は通常カウントされないが、大量移行時は頭に入れておく) |

移行対象がこれを超える規模なら、共有ドライブの分割を先に設計すること。

## 6.2 移行で「維持されるもの」「変わるもの」

検証や社内アナウンスの際に重要な一覧。

| 項目 | 移動されたファイル | 作り直されたフォルダ |
| --- | --- | --- |
| ID / URL | ✅ **そのまま** (ブックマークやリンク集は生きる) | ❌ 変わる (新規作成のため) |
| 版履歴 (変更履歴) | ✅ 維持 | — |
| コメント | ✅ 維持 | — |
| 所有者 | 🔄 共有ドライブ (移行先組織) に変わる — これが目的 | 最初から共有ドライブ |
| 共有設定 | ⚠ 基本は引き継がれるが、以後は共有ドライブのポリシー (メンバー役割) が優先される。移行後に必要な人がアクセスできるか要確認 | メンバー役割に従う |
| スター・「最近使用した」等の個人メタ情報 | ❌ 個人アカウントに紐づくため引き継ぎ対象外 | — |

> ⚠ **コピーで救済されたファイルは別**: ID が変わり、版履歴・コメントは
> 引き継がれない。レポートの「コピー救済」件数を必ず確認すること。

## 6.3 移動できないもの・注意が必要なもの

| 対象 | 挙動 |
| --- | --- |
| **他人がオーナーのファイル** | **処理せずスキップ**し、移動元にそのまま残す。所有権は他人のままで「移行」の意味をなさないため、移動もコピーも行わない。進捗タブの「スキップ（他人所有）」に集計され、**「スキップ一覧」シート**に所有者付きで一覧される (→ [6.6](#66-他人所有ファイルはスキップされる)) |
| **ショートカット** | ショートカット自体は移動されるが、**参照先のファイルは動かない**。参照先が移行元アカウントの別の場所にある場合、将来アクセスできなくなる可能性がある |
| Google マイマップなど一部の特殊タイプ | 共有ドライブ非対応のタイプは移動もコピーも失敗することがある → 失敗一覧に記録されるので個別対応 |
| ゴミ箱の中のアイテム | 対象外 (`trashed = false` で除外) |
| **同一階層に同名フォルダが複数ある場合** | find-or-create の仕様上、移行先では**1つのフォルダに統合 (マージ)** される |
| 移行先に同名フォルダが既にある場合 | そのフォルダを再利用して中へ移行する (マージ)。別名にしたければ先に移行先をリネームしておく |
| フォーム (Google フォーム) | 移動されるが、回答の収集先スプレッドシートとの紐づきは環境により要確認 |

## 6.4 トラブルシューティング

| 症状 / エラー | 原因 | 対処 |
| --- | --- | --- |
| `「設定」シートの必須項目が未入力です` | フォルダ ID の入力漏れ | 「設定」シートの移行元/移行先 ID を入力 ([3.11](./03_setup_guide.md#311-設定シートに入力する)) |
| `「設定」シートがありません` | 設定シート未作成 | メニュー「① 設定シートを準備 / 初期化」を実行 |
| メニュー「📁 ドライブ移行」が出ない | onOpen 未実行 / コード未保存 | スプレッドシートを再読み込み ([3.10](./03_setup_guide.md#310-動作確認とシートメニューの表示)) |
| `移行先 (…) は共有ドライブ内にありません` | 移行先にマイドライブのフォルダを指定した | 共有ドライブ (内のフォルダ) の ID を指定する |
| `File not found` (404) | ID の打ち間違い / そのアカウントからアクセス権がない | URL から ID を取り直す。移行元アカウントで対象を開けるか確認 |
| `insufficientFilePermissions` / `The user does not have sufficient permissions` (403) | 共有ドライブのメンバー役割が弱い (投稿者以下)、またはメンバー未追加 | 「コンテンツ管理者」以上で追加し直す ([3.2](./03_setup_guide.md#32-移行先ユーザー共有ドライブの準備とメンバー追加)) |
| メンバー追加自体ができない | 管理コンソールで組織外ユーザーの参加が許可されていない | [3.1](./03_setup_guide.md#31-移行先の管理者組織外ユーザーの参加を許可する) を管理者に依頼 |
| `userRateLimitExceeded` などが頻発 | API レート制限 | 自動リトライされるので基本放置でよい。止まる場合は時間を置いてメニュー「中断からの再開」 |
| 実行が「エラー3回連続」で止まった (`ERROR`) | ネットワーク断・クォータ枯渇・権限変更など | 「進捗」シートとメール通知で原因を確認 → 対処後メニュー「中断からの再開」 |
| トリガーが動いていない気がする | — | エディタ左の時計アイコン (トリガー) に `resumeMigration` があるか確認。無ければメニュー「中断からの再開」で復帰する |
| `移行状態が大きくなりすぎました` | フォルダ数が極端に多い | サブフォルダごとに移行元フォルダ ID を変えて分割実行 |
| 承認画面で「確認されていません」警告 | 自作スクリプトなので正常 | [3.10](./03_setup_guide.md#310-動作確認とシートメニューの表示) の手順で許可 |

## 6.5 FAQ

**Q. 途中でブラウザを閉じてもいい?**
A. よい。処理は Google のサーバー側で動いており、中断・再開もトリガーが自動で行う。

**Q. 途中経過はどこで見る?**
A. 「進捗」シートに自動更新される。最新化したいときはメニュー「③ 進捗を更新して表示」。
失敗の明細は「失敗一覧」シートに出る。

**Q. 間違えて実行してしまった。巻き戻せる?**
A. 自動の巻き戻し機能はない。移動済みファイルは移行先の共有ドライブに
あるので、手動で (または本ツールの逆方向設定で) 戻すことは可能。
だからこそ**必ず「ドライラン」ON で計画を確認してから本実行**すること。

**Q. 移行中に移行元へファイルが追加されたら?**
A. タイミングにより移行される場合とされない場合がある。完了後にもう一度
メニュー「② 移行を開始」すれば、べき等性 (第4章) により**残っている分だけ**が
移行される。移行期間中は書き込みを止めてもらうのが理想。

**Q. 複数のフォルダを移行したい**
A. 1つ完了するごとに「設定」シートの移行元/移行先 ID を書き換えてメニュー
「② 移行を開始」を実行する (状態は完了時に終わっているので「状態をリセット」は
不要だが、「③ 進捗を更新して表示」で DONE を確認すると安全)。

**Q. 移動元のフォルダに「このフォルダの中身は移動しました.txt」が残るのはなぜ?**
A. 設定「移動元に案内ファイル(.txt)を残す」(既定 ON) の動作。ドメイン間では
フォルダ自体を移動できず移動元に殻が残るため、その中に移行先フォルダへのリンクを
記した案内を置いている。不要なら設定を OFF にする。この案内ファイル自体は移行先へ
移動されず、後片付け(空フォルダ削除)ではフォルダごと片付く。詳細は
[第3章 3.16](./03_setup_guide.md#316-移動元に残る案内ファイル)。

**Q. 移行元が Gmail の無料アカウント (@gmail.com) でも使える?**
A. 使える。仕組みは同一で、この教科書の「@google.com」を読み替えるだけ。
トリガー合計 90分/日 のクォータだけ意識しておく (6.1)。

**Q. 自分がオーナーでないファイルは「失敗」になる?**
A. いいえ。**「スキップ（他人所有）」として別集計**され、「失敗」には含まれない。
移動もコピーもせず移動元に残す。どのファイルを・誰の所有物としてスキップしたかは
「スキップ一覧」シートで確認できる (→ [6.6](#66-他人所有ファイルはスキップされる))。

**Q. 「失敗」の種類を知りたい**
A. 「進捗」シートの「失敗（合計）」の下に種類別の内訳が出る
(移動失敗（コピー救済OFF）/ 移動もコピーも失敗 / フォルダ処理失敗 / その他)。
各明細は「失敗一覧」シートの「種別」列でも確認できる。

**Q. 実行アカウントを移行先アカウントにしてはだめ?**
A. だめではないが劣る。移行先アカウントは移行元ユーザーのファイルを「移動」できず、
本ツールでは他人所有としてすべてスキップされてしまう。詳細は
[第2章 2.3](./02_solution_architecture.md#23-どちらのアカウントで実行するか)。

## 6.6 他人所有ファイルはスキップされる

自分が**所有していない**ファイル (共有されて自分のフォルダに入っているだけの
ファイル等) は、**移動もコピーもせずスキップ**し、移動元にそのまま残す。

理由: 共有ドライブへ移動できるのは原則オーナーだけで、他人所有ファイルを
コピーしても「共有ドライブ所有の別ファイル」ができるだけ (原本の所有者は変わらない)。
これは所有権の「移行」にならないため、勝手に複製せず、**気づけるように記録して残す**
方針にしている。

- 件数は「進捗」シートの **「スキップ（他人所有）」** に集計される
- 明細 (パス / ファイル名 / **所有者** / ファイルID) は **「スキップ一覧」シート**に出る
- 対処例: 所有者本人にそのファイルの移行(または共有ドライブへの配置)を依頼する、
  もしくは所有者からオーナー権限を受け取ってから再実行する

<details>
<summary>📖 なぜ「失敗」ではなく「スキップ」なのか</summary>

「失敗」はツールが本来できるはずの処理に失敗した事象を指す。他人所有ファイルは
仕様上そもそも移行対象にできない (エラーですらなく設計上の対象外) ため、
「失敗」と混ぜず「スキップ」として分けている。こうすることで、進捗タブの
「失敗（合計）」は**本当に対処が必要な異常だけ**を表すようになる。

</details>
