# 第 7 章: スプレッドシート UI の仕組みと共有

本ツールの利用者インターフェースは **スプレッドシート** です。プロジェクトをスプレッドシートに紐付けて(コンテナバインド)デプロイすると、シートのカスタムメニューから実行できるようになります。実体は `src/sheet.ts` です。

この章では「なぜこの形なのか」「共有はどう考えるか」「シート UI のコードはどう動いているか」を解説します。セットアップ手順は[第 2 章](./02-setup.md)、日々の実行手順は[第 5 章](./05-operations.md)を参照してください。

| 特徴 | 内容 |
| --- | --- |
| 設定はセルに書く | 「設定」シートの B2(譲渡先・必須)・B3(対象フォルダ・ツリー走査では必須)・B4(モード)・B5(譲渡方式)。コードを触らない |
| 実行はメニューから | シート上部の「所有権譲渡」メニュー |
| 結果が台帳に残る | 「譲渡ログ」シートに 1 件 1 行で記録(誰が・何を・どこへ・結果)。監査証跡になる |
| 配布 = シート共有 | 使ってほしい人にスプレッドシートを共有するだけ |

<details>
<summary>📘 用語解説: コンテナバインド / スタンドアロン</summary>

Apps Script プロジェクトには 2 形態あります。単体で存在する「スタンドアロン型」と、スプレッドシートなどの「入れ物(コンテナ)」に紐付いた「**コンテナバインド型**」です。バインド型は入れ物の UI(メニューやダイアログ)を拡張でき、`SpreadsheetApp.getActiveSpreadsheet()` で自分の入れ物に直接アクセスできます。本プロジェクトは第 2 章の手順(`clasp create-script --type sheets`)でバインド型として作ります。

</details>

## 7.1 なぜスプレッドシートをインターフェースにするのか

1. **設定ミスが見える**: 設定がセルにあるので、実行前に隣の人とダブルチェックできる。モードはプルダウン(DRY RUN / 本番)で誤入力を防止し、**譲渡先と対象フォルダにはデフォルト値がない**(未入力は必ずエラー)
2. **台帳が残る**: 「何が誰に譲渡されたか」の記録がシートに残り、あとから検索・共有できる。引き継ぎの証跡としてそのまま提出できる
3. **配布が簡単**: URL やデプロイ設定の知識が不要。シートを共有された人がメニューを押すだけ
4. **本人の権限で動く**: メニューから実行した処理は「押した本人」として動くため、対象は常に**本人が所有するファイルだけ**。進捗・停止・再開トリガーも本人専用(第 3 章のユーザープロパティ / ユーザーロック設計がそのまま効く)

## 7.2 シートの構成

メニューの「初期設定」が次の 2 シートを自動で作ります(`sheetSetup()`)。

**「設定」シート** — 実行前に入力する場所

| セル | 内容 |
| --- | --- |
| B2 | 譲渡先メールアドレス(**必須**・デフォルトなし) |
| B3 | 対象フォルダの ID または URL(**ツリー走査では必須**・デフォルトなし。検索走査では使われない) |
| B4 | モード。`DRY RUN(予行演習)` / `本番(実際に譲渡する)` の 2 択プルダウン(初期値は DRY RUN) |
| B5 | 譲渡方式。`直接譲渡(Workspace 同一ドメイン向け)` / `招待方式(個人アカウント向け・ネイティブ形式のみ)` の 2 択プルダウン(初期値は直接譲渡)。詳細は付録 B |

**「譲渡ログ」シート** — 実行結果の台帳

| 列 | 内容 |
| --- | --- |
| 日時 / 実行者 | いつ・誰の実行か(複数人で使っても実行者列で区別できる) |
| 結果 | `DRY RUN 対象` / `譲渡完了` / `エラー` / `サマリ` |
| 種別 / 名前 / ID | 対象アイテムの情報 |
| 譲渡先 / 詳細 | 譲渡先アドレスと、エラー時のメッセージ |

<details>
<summary>📘 用語解説: データの入力規則(プルダウン)</summary>

スプレッドシートのセルに「この値しか入れられない」という制約を付ける機能です。`SpreadsheetApp.newDataValidation()` でスクリプトからも設定でき、本プロジェクトではモード欄を 2 択プルダウンにしています。自由入力による誤設定をここで防ぎます。なお `readSheetSettings()` は、それでも想定外の値が来た場合に**安全側(DRY RUN)に倒す**ようにしています。

</details>

## 7.3 共有範囲の考え方

シート UI の共有範囲は **スプレッドシートの共有設定そのもの**です。Web 公開のような追加の仕組みはありません。

- **シートを共有された人だけ**がメニューを使えます。組織内の後任・チームメンバーへの限定共有と相性が良い形です
- 設定セルを書き換えるため、利用者には**編集者**権限で共有します
- メニューから実行した処理は**押した本人の権限**で動きます。各自が初回に承認画面を通り(第 2 章 2.8)、対象は本人の所有物だけです
- 進捗・停止・再開トリガーは利用者ごとに独立しています(他人の実行を止めることはできません)

注意点が 2 つあります。

1. **設定セルは全員で共有**です。同時に 2 人が使うと「相手の入力した譲渡先」のまま開始してしまう事故がありえます。開始前の確認ダイアログに譲渡先と対象が表示されるので必ず読むこと。同時利用が多い運用なら、時間をずらすルールにするか、利用者ごとにシートを複製してください
2. **台帳シートは全員分が 1 つに記録**されます(実行者列で区別)。他の利用者に見られたくない場合は、シートを複製して個別運用にします

<details>
<summary>📘 用語解説: 編集権限とスクリプト実行の関係</summary>

バインドされたスクリプトは、シートの共有相手なら実行できます(それぞれが初回に自分の承認を行います)。また、シートの**編集者はスクリプトのコードも閲覧・編集できる**点は覚えておいてください。「UI だけ使わせてコードは触らせない」を厳密にやりたい場合は、閲覧権限 + 入力欄の代替手段が必要になり、本プロジェクトの範囲外です(信頼できるメンバーへの共有を前提としています)。

</details>

## 7.4 コードの読みどころ(sheet.ts)

- `onOpen()`: メニューを組み立てるだけの関数。シートを開くたびに自動実行されます
- `sheetSetup()`: 「設定」「譲渡ログ」シートを作る初期化処理。**何度実行しても壊れない(冪等)**ように、既存シートがあれば作り直しません
- `readSheetSettings()`: セルから設定を読み取ります。フォルダ ID は URL のままの貼り付けにも対応(`normalizeFolderIdInput()`)し、モードの値が想定外なら DRY RUN、譲渡方式の値が想定外なら直接譲渡として扱います(いずれも安全側に倒す)
- `sheetStartWithStrategy()`: 入力の検証(B2 必須・ツリー走査なら B3 必須)→ 確認ダイアログ(本番は 2 段階)→ `startTransferWithStrategy()` に実行設定を渡して開始、という入口の門番です
- `recordSheetLog()` / `flushSheetLog()`: 台帳への記録。**1 行ずつ書かずにバッファへためて、バッチの終わりに `setValues()` で一括書き込み**します。シートへの書き込みは 1 回あたり数十ミリ秒かかるため、数千件を 1 件ずつ書くと 6 分制限を圧迫してしまうからです

> 💡 メニューから開始した直後の応答性について: 最初のバッチだけ `CONFIG.uiFirstBatchMs`(45 秒)で切り上げてダイアログを返し、残りはトリガーの自動再開(1 バッチ 4.5 分)に任せます。ダイアログが閉じたあとも処理が続いているのはこのためです。

<details>
<summary>📘 用語解説: 冪等(べきとう)</summary>

「何回実行しても、1 回実行したのと同じ結果になる」性質のことです。初期化処理やリトライされうる処理は冪等に作っておくと、「うっかり 2 回押してしまった」「途中で失敗してやり直した」というときに壊れません。

</details>

<details>
<summary>📘 用語解説: onOpen(シンプルトリガー)</summary>

`onOpen` という名前の関数は、スプレッドシートを開いたときに GAS が自動で呼び出す特別な関数です(シンプルトリガー)。承認前でも動きますが、できることが制限されています(メニュー追加は OK)。本プロジェクトではカスタムメニューの追加だけを行っています。

</details>

## 7.5 マニフェストへの影響

シート UI のために、`appsscript.json` の `oauthScopes` に 2 つのスコープを含めています(第 4 章 4.3 の一覧の一部)。

| スコープ | 用途 |
| --- | --- |
| `spreadsheets.currentonly` | バインド先のシート(設定・台帳)の読み書き。**他のスプレッドシートには触れない**限定スコープ |
| `script.container.ui` | カスタムメニューと確認ダイアログの表示 |
