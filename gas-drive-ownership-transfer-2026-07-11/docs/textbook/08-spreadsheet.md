# 第 8 章: スプレッドシート UI(推奨インターフェース)

このプロジェクトを **スプレッドシートに紐付けて(コンテナバインド)** デプロイすると、シートのカスタムメニューから実行できるようになります。実体は `src/sheet.ts` です。

エンジニア以外の人に配る場合はこのスタイルが最も扱いやすく、本プロジェクトの**推奨インターフェース**です。

| 特徴 | 内容 |
| --- | --- |
| 設定はセルに書く | 「設定」シートの B2(譲渡先)・B3(対象フォルダ)・B4(モード)。コードを触らない |
| 実行はメニューから | シート上部の「所有権譲渡」メニュー |
| 結果が台帳に残る | 「譲渡ログ」シートに 1 件 1 行で記録(誰が・何を・どこへ・結果)。監査証跡になる |
| 配布 = シート共有 | 使ってほしい人にスプレッドシートを共有するだけ |

<details>
<summary>📘 用語解説: コンテナバインド / スタンドアロン</summary>

Apps Script プロジェクトには 2 形態あります。**スタンドアロン**は単体で存在するプロジェクト(第 2 章で作ったもの)。**コンテナバインド**はスプレッドシートやドキュメントなどの「入れ物(コンテナ)」に紐付いたプロジェクトで、入れ物の UI(メニューやダイアログ)を拡張でき、`SpreadsheetApp.getActiveSpreadsheet()` で自分の入れ物に直接アクセスできます。どちらの形態でも、このリポジトリのコードはそのまま動きます(バインドでない場合はシート UI 関連が使えないだけです)。

</details>

## 8.1 なぜスプレッドシート UI が良いのか

1. **設定ミスが見える**: 設定がセルにあるので、実行前に隣の人とダブルチェックできる。モードはプルダウン(DRY RUN / 本番)で誤入力を防止
2. **台帳が残る**: 「何が誰に譲渡されたか」の記録がシートに残り、あとから検索・共有できる。引き継ぎの証跡としてそのまま提出できる
3. **配布が簡単**: Web アプリのような URL 配布・デプロイ設定の理解が不要。シートを共有された人がメニューを押すだけ
4. **本人の権限で動く**: メニューから実行した処理は「押した本人」として動くため、対象は常に**本人が所有するファイルだけ**。進捗・停止・再開トリガーも本人専用(第 3 章のユーザープロパティ / ユーザーロック設計がそのまま効く)

## 8.2 セットアップ

### 方法 A: シートごと新規作成(おすすめ)

`clasp create-script` はスプレッドシート付きのプロジェクトも作れます。

```bash
$ mise run build
$ npx clasp create-script --type sheets --title "Drive 所有権一括譲渡" --rootDir dist
$ mise run push
```

`--type sheets` により、新しいスプレッドシートと、それにバインドされた Apps Script プロジェクトが作られ、`.clasp.json` も自動生成されます。

### 方法 B: 既存のスプレッドシートに紐付ける

1. 対象のスプレッドシートを開き、**拡張機能 → Apps Script** でバインドプロジェクトを作成
2. エディタの「プロジェクトの設定」(⚙)から**スクリプト ID** をコピー
3. `.clasp.json` を作成(`.clasp.json.example` をコピーして ID を書き換え)
4. `mise run push`

### 初期設定

1. スプレッドシートを開き直す(リロード)と、メニューバーに **「所有権譲渡」** が現れます(`onOpen` が動くため数秒かかることがあります)
2. **所有権譲渡 → 初期設定(設定・ログシートを準備)** を実行
   - 初回はここで承認画面が出ます(第 2 章 2.9 と同じ流れ)
3. 「設定」シートと「譲渡ログ」シートが自動で作られます

<details>
<summary>📘 用語解説: onOpen(シンプルトリガー)</summary>

`onOpen` という名前の関数は、スプレッドシートを開いたときに GAS が自動で呼び出す特別な関数です(シンプルトリガー)。承認前でも動きますが、できることが制限されています(メニュー追加は OK)。本プロジェクトではカスタムメニューの追加だけを行っています。

</details>

## 8.3 使い方(運用フロー)

第 5 章の安全フローをシート上で行います。

1. **設定シートに入力**
   - B2: 譲渡先メールアドレス(同じドメインのユーザー)
   - B3: 対象フォルダの ID または URL(空欄 = マイドライブ全体。URL を貼れば ID を自動で取り出します)
   - B4: `DRY RUN(予行演習)` ← まずはこのまま
2. **所有権譲渡 → 所有アイテム数を確認** で規模を把握
3. **所有権譲渡 → 開始(ツリー走査)** → 確認ダイアログで内容をチェックして OK
4. 「譲渡ログ」シートに `DRY RUN 対象` の行が並ぶので、**対象に間違いがないか確認**
   - 長い場合は自動で中断・再開しながら進みます。シートは閉じても OK
   - 進捗は **所有権譲渡 → 進捗を確認** で表示(自分の実行分だけが表示されます)
5. 問題なければ B4 を `本番(実際に譲渡する)` に変えて、もう一度 **開始**
   - 本番は確認ダイアログが **2 段階** になります
6. 完了すると「譲渡ログ」に `サマリ` 行が入ります。`エラー` 行があれば詳細列を確認して対処(第 5 章 5.8)

> 💡 メニュー実行後のダイアログが閉じたあとも、続きの処理はサーバー側のトリガーが自動で進めます。最初のバッチだけ短時間(45 秒)で切り上げて、すぐに操作を返す設計です(第 7 章 7.5 と同じ仕組み)。

## 8.4 共有範囲の考え方(シート版)

Web アプリ(第 7 章)と違い、シート UI の共有範囲は **スプレッドシートの共有設定そのもの**です。

- **シートを共有された人だけ**がメニューを使えます。組織内の後任・チームメンバーへの限定共有と相性が良い形です
- 設定セルを書き換えるため、利用者には**編集者**権限で共有します
- メニューから実行した処理は**押した本人の権限**で動きます。各自が初回に承認画面を通り、対象は本人の所有物だけです
- 進捗・停止・再開トリガーは利用者ごとに独立しています(他人の実行を止めることはできません)

注意点が 2 つあります。

1. **設定セルは全員で共有**です。同時に 2 人が使うと「相手の入力した譲渡先」のまま開始してしまう事故がありえます。開始前の確認ダイアログに譲渡先が表示されるので必ず読むこと。同時利用が多い運用なら、時間をずらすルールにするか、利用者ごとにシートを複製してください
2. **台帳シートは全員分が 1 つに記録**されます(実行者列で区別)。他の利用者に見られたくない場合は、シートを複製して個別運用にします

## 8.5 コードの読みどころ(sheet.ts)

- `onOpen()`: メニューを組み立てるだけの関数。シートを開くたびに自動実行されます
- `sheetSetup()`: 「設定」「譲渡ログ」シートを作る初期化処理。**何度実行しても壊れない(冪等)**ように、既存シートがあれば作り直しません
- `readSheetSettings()`: セルから設定を読み、モードの値が想定外なら**安全側(DRY RUN)に倒します**
- `sheetStartWithStrategy()`: 確認ダイアログ(本番は 2 段階)→ `startTransferWithStrategy()` に入力値を渡して開始。`sheetLog: true` を渡すのがシート UI の印です
- `recordSheetLog()` / `flushSheetLog()`: 台帳への記録。**1 行ずつ書かずにバッファへためて、バッチの終わりに `setValues()` で一括書き込み**します。シートへの書き込みは 1 回あたり数十ミリ秒かかるため、数千件を 1 件ずつ書くと 6 分制限を圧迫してしまうからです

<details>
<summary>📘 用語解説: 冪等(べきとう)</summary>

「何回実行しても、1 回実行したのと同じ結果になる」性質のことです。初期化処理やリトライされうる処理は冪等に作っておくと、「うっかり 2 回押してしまった」「途中で失敗してやり直した」というときに壊れません。

</details>

<details>
<summary>📘 用語解説: データの入力規則(プルダウン)</summary>

スプレッドシートのセルに「この値しか入れられない」という制約を付ける機能です。`SpreadsheetApp.newDataValidation()` でスクリプトからも設定でき、本プロジェクトではモード欄を `DRY RUN(予行演習)` / `本番(実際に譲渡する)` の 2 択プルダウンにしています。自由入力による誤設定をここで防ぎます。

</details>

## 8.6 マニフェストへの影響

シート UI のために、`appsscript.json` の `oauthScopes` に 2 つ追加しています。

| スコープ | 用途 |
| --- | --- |
| `spreadsheets.currentonly` | バインド先のシート(設定・台帳)の読み書き。**他のスプレッドシートには触れない**限定スコープ |
| `script.container.ui` | カスタムメニューと確認ダイアログの表示 |
