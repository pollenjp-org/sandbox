# 📚 Textbook: Google Drive 所有権一括譲渡ツール

このディレクトリは、本プロジェクト(Google Drive 内で自分が所有するファイル/フォルダを、指定したユーザーへ再帰的に一括譲渡する Google Apps Script)を **新規参画者がゼロから理解できるようにするための教科書** です。

## 対象読者

- このプロジェクトに初めて触れる人
- Google Apps Script(GAS)や clasp を使ったことがない人
- 「コードは読めるが、なぜこういう設計になっているのか知りたい」人

プログラミングの基礎(変数・関数・if/while がわかる程度)以外の前提知識は仮定しません。専門用語が出てきた箇所には、その直下に折りたたみ(アコーディオン)形式の用語解説を挿入しています。

<details>
<summary>📘 用語解説: アコーディオン(折りたたみ)</summary>

クリックすると開閉する UI 部品のことです。この教科書では GitHub の Markdown がサポートする `<details>` タグを使っており、▶ マークをクリックすると解説が展開されます。用語をすでに知っている人は読み飛ばせるようになっています。

## 章構成(この順に読むことを推奨)

| 章 | ファイル | 内容 |
| --- | --- | --- |
| 第 1 章 | [01-overview.md](./01-overview.md) | このツールは何か。解決する課題、全体像、重要な前提と制約 |
| 第 2 章 | [02-setup.md](./02-setup.md) | 開発環境の構築から、自分の Apps Script プロジェクトへの初回デプロイまで |
| 第 3 章 | [03-architecture.md](./03-architecture.md) | 設計の解説。6 分制限との戦い方(バッチ処理・チェックポイント・自動再開) |
| 第 4 章 | [04-code-walkthrough.md](./04-code-walkthrough.md) | ソースコードをファイルごと・関数ごとに読み解く |
| 第 5 章 | [05-operations.md](./05-operations.md) | 実行手順(運用マニュアル)。シートのメニューで DRY RUN → 本番実行 → 完了確認、トラブルシューティング |
| 付録 | [06-appendix.md](./06-appendix.md) | 発展的な話題。個人アカウントの場合、図の再生成方法(PlantUML / drawio)、参考リンク |
| 第 7 章 | [07-spreadsheet.md](./07-spreadsheet.md) | スプレッドシート UI の仕組みと共有。設定セル・台帳シート・共有範囲の考え方、sheet.ts のコード解説 |

利用者インターフェースは**スプレッドシート**の 1 つだけです。設定は「設定」シートのセル、実行はカスタムメニュー、結果は「譲渡ログ」シート(台帳)で完結します。

## 表記ルール

- `このように囲まれた文字` はコマンド・ファイル名・コードを表します
- コマンド例の先頭の `$` は入力しません(シェルのプロンプトを表す慣習です)
- ⚠️ が付いた枠は「読み飛ばすと事故につながる」重要な注意です

> ⚠️ このツールは **ファイルの所有権を実際に変更する** ものです。必ず第 5 章の手順どおり、DRY RUN(予行演習モード)での確認を経てから本番実行してください。
